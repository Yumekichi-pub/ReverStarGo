🚫 起きてはダメ台帳（ANTI_LEDGER）

このファイルは、ReverStarGo の本体プログラムが動いている間に 絶対に発生してはいけない事象 を 1 箇所にまとめた台帳です。「正リスト（EVENT_LEDGER）」が「起きるべきこと」を見張るのに対し、こちらは「起きてはダメなこと」を見張ります。

使い方

  • 常時監視: チェッカ A（イベントチェッカ）とチェッカ B（ゼウスへの道チェッカ）が動いている間、ここにある全項目がバックグラウンドで常に見張られます
  • 一度でも発生したら即 ❌: 通常イベントと違い、こちらは「1 回でも起きたらアウト」の判定です
  • 追加方法: 新しい禁止事象が見つかったら、該当カテゴリの表に 1 行追加するだけ
  • 削除方法: 行を削除するか、行頭に ~~ を付けて打ち消し線にする
  • ID 命名規則: ANTI_連番 形式

検出の仕組み（実装方針メモ）

ANTI チェックは、event_check.html や zeus_path_check.html の起動時に 共通のウォッチドッグスクリプト を iframe 内の本体ゲームに注入することで実現します。具体的には：

  1. State assertion 系（ANTI_006, 007, 008, 009 など）
    • 各 executeMove / applyMove の 後 に hook を挟み、盤面状態を検査
  2. Function hook 系（ANTI_001, 002, 011, 012, 013, 014, 015, 016, 017, 020 など）
    • 該当関数を差し替え、呼び出し時の引数や状態を記録
  3. Browser-level hook 系（ANTI_018, 019 など）
    • window.onerror / MutationObserver などブラウザ機能を利用

これにより、本体 index.html には一切手を加えず、チェッカ側からの「外付け監視」だけで全 20 項目を検出できます。
